test_debug: debug 機能のテストをするディレクトリ

DLOG() などは，cerr に出力する機能なので，assert でテストするのは難しい．
以下のようにする:

* ソースコードは test?_skel.cc に書く．(? は 1, 2, 3, ... など)
  test1_skel.cc をまねて書くと良い．

* 次のように出力を行う:

    {{{{ test_1.cc:42              <<- 行頭の「{{{{」がマークで，その後はラベル
    a=[1, 2, 3, ]                  <<- DLOGKなどの実際の結果
    ----
    a=[1, 2, 3]                    <<- 期待する出力
    }}}}

* 上の出力を行うためのマクロが用意されている．(test1_skel.cc を参照)
  * たいてい，CMPR("a=[1, 2, 3]", DLOGK(a)); というように書けば良い．
  * ソースが2行以上に渡る場合などは，次のようにする:
      CMPR_BEGIN();
      ... DLOGK など ...
      CMPR_SEP();
      cerr << ... 期待する出力 ... << endl;
      CMPR_END();

* make を行うと，libins と g++ が走って，test?_t ができる．

* run というスクリプトがある．これを走らせると，上のように期待と実際が異なった場合に報告する．
  対象プログラムは，test?_t．
